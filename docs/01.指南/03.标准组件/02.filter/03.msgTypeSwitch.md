`msgTypeSwitch`组件：消息路由。根据传入的消息类型（msgType）路由到一个或多个输出链。

## 配置

无

## Relation Type

通过`msgType`与下一个或多个节点连接

## 执行结果

该组件不会改变`msg`、`metadata`和`msgType`内容。

## 配置示例

```json
{
  "id": "s1",
  "type": "msgTypeSwitch",
  "name": "过滤"
}
```

## 应用示例

通过`msgType`路由到不同的节点进行处理。
```json
{
  "ruleChain": {
    "id":"rule01",
    "name": "测试规则链-msgTypeSwitch",
    "root": true
  },
  "metadata": {
    "nodes": [
      {
        "id": "s1",
        "type": "msgTypeSwitch",
        "name": "过滤"
      },
      {
        "id": "s2",
        "type": "log",
        "name": "记录日志1",
        "configuration": {
          "jsScript": "return 'handle msgType='+ msgType+':s2';"
        }
      },
      {
        "id": "s3",
        "type": "log",
        "name": "记录日志2",
        "configuration": {
          "jsScript": "return 'handle msgType='+ msgType+':s3';"
        }
      },
      {
        "id": "s4",
        "type": "log",
        "name": "记录日志3",
        "configuration": {
          "jsScript": "return 'handle msgType='+ msgType+':s4';"
        }
      }
    ],
    "connections": [
      {
        "fromId": "s1",
        "toId": "s2",
        "type": "TEST_MSG_TYPE1"
      },
      {
        "fromId": "s1",
        "toId": "s3",
        "type": "TEST_MSG_TYPE1"
      },
      {
        "fromId": "s1",
        "toId": "s4",
        "type": "TEST_MSG_TYPE2"
      }
    ],
    "ruleChainConnections": null
  }
}
```
